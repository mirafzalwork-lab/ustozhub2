# 💬 Система Обмена Сообщениями - Полная Документация

## ✅ Реализованные Возможности

### 📋 Основной Функционал

#### 1. **Модели Базы Данных**
- ✅ **Conversation** - модель переписки между учителем и учеником
  - Связь один-к-одному между учителем и учеником
  - Отслеживание активности переписки
  - Временные метки создания и обновления
  
- ✅ **Message** - модель сообщения
  - Связь с перепиской
  - Отправитель (User)
  - Содержание сообщения (до 2000 символов)
  - Статус прочитанности (is_read)
  - Временная метка

#### 2. **Views (Представления)**

**a) conversations_list** - список всех переписок
- Показывает все активные переписки пользователя
- Отображает последнее сообщение
- Показывает счётчик непрочитанных сообщений
- Сортировка по времени последнего обновления

**b) conversation_detail** - детальная страница переписки
- Отображение всех сообщений в хронологическом порядке
- Форма для отправки новых сообщений
- Автоматическая отметка сообщений как прочитанных
- Возможность удаления переписки

**c) start_conversation** - начать переписку
- Создание новой переписки с пользователем
- Проверка прав доступа (учителя могут писать ученикам и наоборот)
- Автоматический переход к существующей переписке, если она уже есть

**d) send_message_ajax** - отправка сообщения (AJAX)
- Асинхронная отправка сообщений
- Валидация данных
- Обновление времени переписки

**e) mark_messages_read** - отметить как прочитанные (AJAX)
- Массовая отметка непрочитанных сообщений

**f) delete_conversation** - удаление переписки
- Деактивация переписки (мягкое удаление)

#### 3. **Context Processor**

**unread_messages_count** - глобальный счётчик непрочитанных
- Доступен во всех шаблонах как `{{ unread_messages_count }}`
- Подсчитывает непрочитанные сообщения для текущего пользователя
- Работает для учителей и учеников

#### 4. **Шаблоны (Templates)**

**a) conversations_list.html**
- Красивый список переписок
- Аватары пользователей
- Превью последнего сообщения
- Бейджи с количеством непрочитанных
- Время последнего сообщения
- Пустое состояние для новых пользователей

**b) conversation_detail.html**
- Чат-интерфейс с сообщениями
- Разделение на "свои" и "чужие" сообщения
- Форма отправки с textarea
- Автопрокрутка к последнему сообщению
- Информация о собеседнике
- Кнопка удаления переписки

**c) base.html (навбар)**
- Кнопка "Сообщения" с иконкой
- Красный бейдж с количеством непрочитанных
- Адаптивный дизайн для мобильных

**d) teacher_detail.html**
- Кнопка "Написать сообщение" для учеников
- Доступна только авторизованным пользователям
- Скрыта для самого учителя

**e) student_detail.html**
- Кнопка "Отправить сообщение" для учителей
- Ограничение: ученики не могут писать другим ученикам

#### 5. **Формы**

**MessageForm** - форма отправки сообщения
- Textarea для ввода текста
- Валидация длины (1-2000 символов)
- Очистка пустых сообщений

#### 6. **URL-маршруты**

```python
# Список всех переписок
/messages/

# Детальная страница переписки
/messages/<uuid:conversation_id>/

# Начать новую переписку
/messages/start/<int:user_id>/

# AJAX отправка сообщения
/api/messages/<uuid:conversation_id>/send/

# AJAX отметить как прочитанные
/api/messages/<uuid:conversation_id>/read/

# Удалить переписку
/messages/<uuid:conversation_id>/delete/
```

## 🔐 Безопасность и Права Доступа

### Проверки Доступа

1. **Только авторизованные пользователи** могут:
   - Просматривать переписки
   - Отправлять сообщения
   - Удалять переписки

2. **Учителя могут**:
   - Писать ученикам
   - Просматривать только свои переписки

3. **Ученики могут**:
   - Писать учителям
   - НЕ могут писать другим ученикам
   - Просматривать только свои переписки

4. **Пользователь НЕ может**:
   - Писать самому себе
   - Просматривать чужие переписки
   - Редактировать чужие сообщения

## 📊 База Данных

### Структура Таблиц

**Conversation**
```
id (UUID, Primary Key)
teacher (FK -> TeacherProfile)
student (FK -> User)
subject (FK -> Subject, nullable)
is_active (Boolean)
created_at (DateTime)
updated_at (DateTime)
```

**Message**
```
id (Integer, Primary Key)
conversation (FK -> Conversation)
sender (FK -> User)
content (Text, max 2000)
is_read (Boolean)
created_at (DateTime)
```

### Индексы и Оптимизация

- Unique constraint на пару (teacher, student) в Conversation
- Сортировка по updated_at для быстрого поиска
- Select related для минимизации запросов

## 🚀 Использование

### Для Учеников

1. **Найти учителя**
   - Перейти на профиль учителя
   - Нажать "Написать сообщение"

2. **Просмотр сообщений**
   - Кликнуть на иконку письма в navbar
   - Увидеть счётчик непрочитанных

3. **Отправить сообщение**

   - Перейти в раздел "Ученики"

2. **Ответить на сообщение**
   - Открыть раздел "Сообщения"
   - Выбрать переписку с непрочитанными (красный бейдж)
   - Прочитать и ответить

3. **Управление переписками**
   - Удалить ненужные переписки
   - Сортировка по активности

## 🔧 Технические Детали

### Технологии

- **Backend**: Django 5.x
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **База данных**: SQLite (dev), PostgreSQL (prod ready)
- **AJAX**: Fetch API для асинхронных запросов

### Производительность

1. **Оптимизация запросов**
   - select_related для ForeignKey
   - prefetch_related для ManyToMany
   - Подсчёт в одном запросе

2. **Кэширование**
   - Context processor кэширует результаты
   - Минимальное количество запросов к БД

### Масштабируемость

- UUID для conversation ID (готово к распределённой системе)
- Мягкое удаление (is_active) для сохранения истории
- Готово к интеграции с WebSocket/Channels для real-time

## 📝 Примеры Использования

### Начать переписку из Python

```python
from teachers.models import User, TeacherProfile, Conversation

teacher = TeacherProfile.objects.get(id=1)
student = User.objects.get(id=2)

conversation, created = Conversation.objects.get_or_create(
    teacher=teacher,
    student=student
)
```

### Отправить сообщение

```python
from teachers.models import Message

message = Message.objects.create(
    conversation=conversation,
    sender=student,
    content="Здравствуйте, хочу записаться на урок!"
)
```

### Получить непрочитанные

```python
unread = Message.objects.filter(
    conversation__student=student,
    is_read=False
).exclude(sender=student).count()
```

## 🐛 Известные Ограничения

1. **Нет real-time обновлений**
   - Сообщения обновляются только при перезагрузке
   - Решение: добавить Django Channels

2. **Нет вложений**
   - Только текстовые сообщения
   - Решение: добавить FileField в Message

3. **Нет групповых чатов**
   - Только 1-на-1 переписки
   - Решение: изменить модель Conversation

4. **Нет уведомлений по email**
   - Пользователь не узнает о новых сообщениях вне сайта
   - Решение: интеграция с Celery + email

## ✅ Тестирование

### Сценарии для Проверки

1. **Ученик пишет учителю**
   - ✅ Создаётся новая переписка
   - ✅ Сообщение отображается
   - ✅ Счётчик у учителя увеличивается

   - ✅ Показывается предупреждение

5. **Удаление переписки**
   - ✅ Переписка скрывается из списка
   - ✅ Данные сохраняются в БД (is_active=False)
