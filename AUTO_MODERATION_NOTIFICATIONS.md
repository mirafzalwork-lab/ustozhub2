# 🎓 Автоматические уведомления для учителей при модерации

## Описание функциональности

При одобрении или отклонении профиля учителя администратором, система **автоматически** создаёт персонализированное уведомление для этого учителя.

---

## 🔔 Типы уведомлений

### 1. **Одобрение профиля** ✅

**Когда:** Администратор одобряет профиль учителя (moderation_status = 'approved')

**Заголовок:** "✅ Ваш профиль одобрен!"

**Текст:**
```
Здравствуйте, [Имя учителя]!

Рады сообщить вам, что ваш профиль учителя успешно прошёл модерацию и был одобрен администратором [Имя админа].

Теперь ваш профиль виден всем пользователям платформы, и ученики смогут находить вас и связываться с вами!

🎉 Желаем вам успехов в преподавании и много благодарных учеников!

Рекомендации для успешного старта:
• Регулярно проверяйте сообщения от учеников
• Отвечайте оперативно на запросы
• Поддерживайте актуальность информации в профиле
• Будьте пунктуальны и профессиональны

С уважением,
Команда UstozHub
```

**Приоритет:** 10 (высокий)

---

### 2. **Отклонение профиля** ❌

**Когда:** Администратор отклоняет профиль учителя (moderation_status = 'rejected')

**Заголовок:** "❌ Профиль не одобрен"

**Текст:**
```
Здравствуйте, [Имя учителя]!

К сожалению, ваш профиль учителя не прошёл модерацию.

[Причина отклонения от модератора, если указана]

Что делать дальше?
• Внимательно изучите комментарий модератора
• Исправьте указанные недостатки в профиле
• Обновите информацию и отправьте профиль на повторную проверку
• При необходимости обратитесь в службу поддержки

Мы всегда рады видеть качественных преподавателей на нашей платформе!

С уважением,
Команда UstozHub
```

**Приоритет:** 10 (высокий)

## 🛠 Техническая реализация

### Изменения в модели `Notification`

Добавлено новое поле для персонализированных уведомлений:

```python
target_user = models.ForeignKey(
    User,
    on_delete=models.CASCADE,
    null=True,
    blank=True,
    related_name='personal_notifications',
    verbose_name='Конкретный пользователь'
)
```

Новый выбор в `TARGET_CHOICES`:
```python
('specific_user', 'Конкретный пользователь'),
```

### Методы `approve()` и `reject()` в `TeacherProfile`

Оба метода теперь автоматически создают уведомления:

```python
def approve(self, moderator, comment=''):
    # ... одобрение профиля ...
    
    # Создаём персонализированное уведомление
    Notification.objects.create(
        title="✅ Ваш профиль одобрен!",
        short_text="...",
        full_text="...",
        target='specific_user',
        target_user=self.user,  # Конкретный учитель
        is_active=True,
        priority=10,
        created_by=moderator
    )
```

## 🔒 Безопасность

### 1. **Персонализация**
- Уведомление видит **только** конкретный учитель
- Используется `target='specific_user'` + `target_user=self.user`
- Другие пользователи не увидят это уведомление

### 2. **Обработка ошибок**
- Создание уведомления обёрнуто в `try-except`
- Если создание уведомления не удалось, процесс модерации **не прерывается**
- Ошибки логируются для отладки

### 3. **Логирование**
```python
logger.info(f"Approval notification created for teacher: {self.user.username}")
logger.error(f"Failed to create approval notification: {e}", exc_info=True)
```

## 📊 Как это работает

### Сценарий 1: Одобрение профиля

1. **Админ** открывает профиль учителя в Django Admin
2. **Админ** нажимает кнопку "Одобрить" (или использует действие)
3. **Система:**
   - Устанавливает `moderation_status = 'approved'`
   - Сохраняет комментарий модератора (если есть)
   - **Автоматически создаёт уведомление** для учителя
4. **Учитель:**
   - Видит колокольчик 🔔 с badge (1)
   - Открывает уведомления
   - Читает поздравление об одобрении

### Сценарий 2: Отклонение профиля

1. **Админ** отклоняет профиль с комментарием
2. **Система** создаёт уведомление с причиной отклонения
3. **Учитель** получает детальную информацию что нужно исправить

## 🎯 Преимущества

✅ **Автоматизация** - уведомления создаются автоматически, админу не нужно ничего делать дополнительно

✅ **Персонализация** - каждый учитель получает своё уникальное уведомление

✅ **Безопасность** - уведомление видит только получатель

✅ **Информативность** - учитель получает полную информацию о статусе модерации

✅ **Надёжность** - ошибки не прерывают процесс модерации

✅ **Логирование** - все действия логируются для аудита

## 🧪 Тестирование

### Тест 1: Одобрение профиля

1. Создайте тестового учителя
2. Зарегистрируйте его (профиль будет pending)
3. Войдите в админку под суперпользователем
4. Одобрите профиль учителя
5. **Проверьте:**
   - Статус изменился на 'approved'
   - В разделе "Уведомления" создано новое уведомление
   - target = 'specific_user'
   - target_user = этот учитель
   - Войдите под учителем
   - Увидьте колокольчик с badge
   - Откройте уведомление и прочитайте

### Тест 2: Отклонение профиля

1. Создайте другого тестового учителя
2. Отклоните его профиль с комментарием "Недостаточно информации"
3. **Проверьте:**
   - Создано уведомление об отклонении
   - В тексте есть комментарий модератора
   - Учитель видит это уведомление

### Тест 3: Приватность

1. Одобрите учителя A
2. Войдите под учителем B
3. **Проверьте:**
   - Учитель B **не видит** уведомление учителя A
   - Количество непрочитанных = 0 (если у B нет своих уведомлений)

## 🔄 Обратная совместимость

Все существующие уведомления продолжают работать:
- `target='all'` - все пользователи
- `target='students'` - только студенты
- `target='teachers'` - все учителя
- `target='admins'` - только админы
- **NEW** `target='specific_user'` + `target_user` - конкретный пользователь

---

## 📌 Важные замечания

1. **Комментарий модератора** автоматически добавляется в текст уведомления при отклонении

2. **Имя модератора** отображается в уведомлении при одобрении

3. **Высокий приоритет** (10) - эти уведомления отображаются выше обычных

4. **Автоматическая очистка** - при удалении пользователя, его персональные уведомления тоже удаляются (CASCADE)
