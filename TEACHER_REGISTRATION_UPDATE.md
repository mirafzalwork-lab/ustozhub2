### 2. API интеграция

Страница использует 3 API endpoint:

```javascript
// Загрузка категорий
GET /api/subjects/categories/
// Ответ: { categories: [ { id, name, icon, color, order, subject_count } ] }

// Загрузка популярных предметов
GET /api/subjects/popular/
// Ответ: { subjects: [ { id, name, category_name, category_icon, category_color } ] }

// Автодополнение (поиск)
GET /api/subjects/autocomplete/?q=матем
GET /api/subjects/autocomplete/?category=1
// Ответ: { subjects: [ ... ] }
```

## 🎯 Как это работает

### Поток выбора предмета:

1. **Пользователь видит 3 варианта выбора:**
   - Клик по категории → показывает предметы этой категории
   - Поиск по названию → живые результаты
   - Клик по популярному предмету → мгновенный выбор

2. **После выбора:**
   - Значение устанавливается в скрытый `<select>`
   - Показывается визуальный badge с названием
   - Форма поиска очищается
   - Категории сбрасываются

3. **Валидация:**
   - Проверяет только видимые карточки
   - Требует хотя бы 1 предмет с ценой
   - Показывает конкретные ошибки для каждой карточки

## 📋 Технические детали

### Совместимость с формами Django

Оригинальные `<select>` элементы скрыты, но продолжают работать:

```html
<!-- Скрытый оригинальный select -->
<div style="display: none;">
    {{ form.subject_1 }}
</div>
```

JavaScript устанавливает значения через:
```javascript
document.getElementById('id_subject_1').value = subjectId;
```

### Debounce для производительности

Поиск использует debounce чтобы не нагружать сервер:

```javascript
searchTimeout = setTimeout(() => {
    searchSubjects(cardNum, query);
}, 300);
```

### Динамическая инициализация

Когда добавляется новая карточка предмета:

```javascript
function addSubject() {
    // ... показать карточку ...
    
    // Инициализировать UI для новой карточки
    renderCategories(activeSubjects);
    renderPopularSubjects(activeSubjects);
    initializeSubjectCard(activeSubjects);
}
```
