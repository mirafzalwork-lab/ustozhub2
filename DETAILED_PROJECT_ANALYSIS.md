## 1. ОБЩАЯ ИНФОРМАЦИЯ О ПРОЕКТЕ

### 1.1 Назначение
**TeacherHub** (UstozHub) — платформа для поиска и связи учителей и учеников в Узбекистане. Платформа предоставляет:

- **Для учителей:**
  - Создание профиля с указанием предметов и цен
  - Модерация профилей администраторами
  - Получение уведомлений о сообщениях через Telegram
  - Отслеживание статистики просмотров профиля
  - Система отзывов и рейтингов

- **Для учеников:**
  - Поиск учителей по различным критериям
  - Просмотр детальной информации о учителях
  - Система избранных учителей
  - Создание собственного профиля с указанием желаемых предметов
  - Получение уведомлений через Telegram

### 1.2 Технологический стек

#### Backend:
- **Django 4.2.0** — основной фреймворк
- **Python 3.11+** — язык программирования
- **SQLite3** — база данных (для разработки, рекомендован PostgreSQL для продакшена)

#### Frontend:
- **Django Templates** — система шаблонов
- **HTML/CSS** — верстка
- **JavaScript** (базовый) — для интерактивности

#### Интеграции:
- **python-telegram-bot 21.0.1** — Telegram бот API
- **Pillow >=12.0.0** — обработка изображений
- **WhiteNoise 6.5.0** — раздача статики

#### Инфраструктура:
- **Pipenv** — управление зависимостями
- **ASGI/WSGI** — развертывание

## 2. АРХИТЕКТУРА И СТРУКТУРА

### 2.1 Архитектурный подход

Проект использует классическую **MVC (Model-View-Controller)** архитектуру Django:

- **Models** — определение структуры данных и бизнес-логики
- **Views** — обработка HTTP запросов и генерация ответов
- **Templates** — отображение данных пользователю
- **Forms** — валидация и обработка пользовательского ввода

### 2.2 Международная локализация (i18n)

Проект поддерживает **3 языка**:
- **Русский** (ru) — язык по умолчанию
- **Узбекский** (uz) — O'zbekcha
- **Английский** (en) — English

**Реализация:**
- Middleware: `LocaleMiddleware` правильно размещен
- URL patterns: `i18n_patterns()` с префиксами языков
- Templates: использование `{% load i18n %}`
- Локализация: файлы `.po` и `.mo` в `locale/`

### 2.3 URL структура

```python
/                       # Главная (список учителей)
/students/              # Список учеников
/teacher/<id>/          # Детальная страница учителя
/student/<id>/          # Детальная страница ученика
/login/                 # Вход
/register/              # Регистрация учителя
/register/student/      # Регистрация ученика
/profile/               # Профиль пользователя
/api/telegram/auth/     # Telegram WebApp авторизация
```

## 3. МОДЕЛИ ДАННЫХ

### 3.1 Основные модели

#### **User (Пользователь)**
Расширенная модель пользователя Django:
- `user_type` — тип пользователя (student/teacher)
- `phone`, `age`, `avatar` — дополнительная информация
- `is_verified` — статус верификации
- Автоматическое изменение размера аватара при сохранении

#### **TeacherProfile (Профиль учителя)**
**Поля:**
- Основная информация: `bio`, `education_level`, `university`, `specialization`
- Опыт: `experience_years` (0-50 лет)
- Предметы: Many-to-Many через `TeacherSubject` (с ценами)
- Локация: `city` (ForeignKey)
- Формат: `teaching_format` (online/offline/both)
- Контакты: `telegram`, `whatsapp`
- Языки преподавания: `teaching_languages` (строка с кодами)
- Время работы: `available_from`, `available_to`, `available_weekdays`
- Рейтинг: `rating`, `total_reviews`, `total_students`
- Модерация: `moderation_status` (pending/approved/rejected)
- Статус: `is_active`, `is_featured`

**Методы:**
- `approve()` / `reject()` — модерация профиля
- `get_views_count()` — статистика просмотров
- `get_unique_viewers_count()` — уникальные посетители
- `get_teaching_languages_display()` — отображение языков

**Оптимизация:**
- 5 индексов для ускорения поиска
- `select_related()` для избежания N+1 запросов

#### **StudentProfile (Профиль ученика)**
**Поля:**
- Образование: `education_level`, `school_university`
- Локация: `city`
- Предметы: `interests`, `desired_subjects` (Many-to-Many)
- Бюджет: `budget_min`, `budget_max`
- Формат: `learning_format`
- Контакты: `telegram`, `whatsapp`
- Доступность: `available_weekdays`
- Статус: `is_active`

**Методы:**
- `get_budget_display()` — форматирование бюджета
- `get_views_count()` — статистика просмотров

**Оптимизация:**
- 5 индексов для фильтрации

#### **TeacherSubject (Предмет учителя)**
Промежуточная модель с дополнительными данными:
- `hourly_rate` — цена за час в сумах
- `is_free_trial` — бесплатное пробное занятие
- `description` — дополнительная информация

**Оптимизация:**
- 2 индекса для фильтрации по цене и предмету

#### **TelegramUser (Telegram пользователь)**
Интеграция с Telegram:
- `telegram_id` — уникальный ID в Telegram
- `telegram_username` — username
- `user` — связь с Django User (опционально)
- `notifications_enabled` — включены ли уведомления
- `started_bot` — нажал ли /start

**Оптимизация:**
- 2 индекса для быстрого поиска

#### **Другие модели:**
- `Subject` — предметы обучения
- `City` — города
- `Certificate` — сертификаты учителей
- `Review` — отзывы о учителях
- `Favorite` — избранные учителя (для учеников)
- `FavoriteStudent` — избранные ученики (для учителей)
- `Conversation` / `Message` — система сообщений
- `ProfileView` — отслеживание просмотров профилей

### 3.2 Связи между моделями

```
User ←──→ TeacherProfile (OneToOne)
User ←──→ StudentProfile (OneToOne)
User ←──→ TelegramUser (OneToOne)

TeacherProfile ←──→ Subject (ManyToMany через TeacherSubject)
StudentProfile ←──→ Subject (ManyToMany для desired_subjects)

TeacherProfile ←──→ City (ForeignKey)
StudentProfile ←──→ City (ForeignKey)

TeacherProfile ←──→ Review (ForeignKey)
TeacherProfile ←──→ Favorite (ForeignKey)
```

## 4. ФУНКЦИОНАЛЬНОСТЬ

### 4.1 Регистрация и аутентификация

#### Регистрация учителя (3 шага):
1. **Шаг 1** — Основная информация:
   - Личные данные (имя, фамилия, email, телефон, возраст, аватар)
   - Профессиональная информация (био, образование, опыт)
   - Локация и формат обучения
   - Время работы и языки преподавания

2. **Шаг 2** — Предметы и цены:
   - До 5 предметов
   - Цена за час для каждого предмета
   - Опция бесплатного пробного занятия
   - Описание по каждому предмету

3. **Шаг 3** — Сертификаты (опционально):
   - Загрузка сертификатов
   - Можно добавить несколько
   - Можно пропустить

#### Регистрация ученика (1 шаг):
- Личные данные
- Выбор интересующих предметов (до 10)
- Указание бюджета (мин/макс)
- Формат обучения
- Доступные дни

#### Авторизация:
- Вход по username или email
- Опция "Запомнить меня"
- Редирект на профиль после входа

### 4.2 Поиск и фильтрация

#### Поиск учителей (`home()`):
**Фильтры:**
- По предмету
- По городу
- По формату обучения (онлайн/офлайн)
- По цене (мин/макс)
- По рейтингу (мин)
- По опыту (мин лет)
- Текстовый поиск (имя, фамилия, био)

**Сортировка:**
- По рейтингу (убывание)
- По дате создания (новые сначала)

**Пагинация:**
- 12 учителей на страницу
- Сохранение фильтров при переключении страниц

#### Поиск учеников (`students_list()`):
**Фильтры:**
- По предмету
- По городу
- По формату обучения
- По уровню образования
- По бюджету (мин/макс)
- Текстовый поиск

**Пагинация:**
- 12 учеников на страницу

### 4.3 Профили и редактирование

#### Просмотр профиля:
- Детальная информация о пользователе
- Статистика просмотров (всего, за день, за неделю, уникальных)
- Список предметов с ценами
- Отзывы и рейтинги (для учителей)
- Похожие учителя/ученики

#### Редактирование профиля:
- Раздельные формы для User и Profile
- Валидация данных
- Сохранение изменений
- Автоматическое обновление связанных данных

### 4.4 Система избранного

- **Для учеников:** добавление учителей в избранное
- **Для учителей:** добавление учеников в избранное
- Отдельные страницы для просмотра избранного
- AJAX-переключение статуса избранного

### 4.5 Модерация профилей учителей

**Статусы:**
- `pending` — на модерации
- `approved` — одобрено
- `rejected` — отклонено

**Функции в админке:**
- Одобрение/отклонение профилей
- Добавление комментариев модератора
- Отслеживание даты и модератора

**Процесс:**
1. Учитель создает профиль → статус `pending`, `is_active=False`
2. Администратор проверяет и одобряет → статус `approved`, `is_active=True`
3. Профиль появляется в поиске

### 4.6 Отслеживание просмотров

**Модель ProfileView:**
- Запись каждого просмотра профиля
- Хранение IP адреса и User Agent
- Разделение по типам профилей (teacher/student)
- Методы для получения статистики:
  - `get_views_count(period)` — просмотры за период
  - `get_unique_viewers_count(period)` — уникальные посетители

**Использование:**
- Автоматическая запись при просмотре детальной страницы
- Исключение просмотров своего собственного профиля
- Отображение статистики в админке

## 5. ИНТЕГРАЦИИ

### 5.1 Telegram Bot Integration

#### Компоненты:

1. **`telegram_bot/bot.py`** — основной бот:
   - Команды: `/start`, `/help`, `/profile`, `/notifications`
   - WebApp кнопка для открытия сайта
   - Создание/обновление TelegramUser в БД
   - Управление уведомлениями

2. **`telegram_bot/notifications.py`** — система уведомлений:
   - `TelegramNotificationService` — сервис отправки
   - Уведомления о новых сообщениях
   - Массовые рассылки
   - Синхронные и асинхронные методы

3. **`teachers/telegram_views.py`** — API для WebApp:
   - `telegram_auth` — аутентификация через Telegram WebApp
   - `link_telegram_account` — привязка аккаунта
   - `telegram_status` — проверка статуса подключения
   - Проверка подлинности данных через HMAC

4. **`teachers/admin_telegram_service.py`** — админ рассылки:
   - Отправка выбранным пользователям
   - Массовая рассылка всем
   - Фильтрация по типу пользователя
   - Детальная статистика отправки

#### Сигналы:
- Автоматическая отправка уведомлений при создании нового сообщения
- Интеграция через Django signals

### 5.2 Telegram WebApp

**Функциональность:**
- Авторизация через Telegram без пароля
- Автоматическое создание/обновление пользователя
- Привязка Telegram аккаунта к Django пользователю
- Проверка подлинности данных через HMAC-SHA256

**Процесс:**
1. Пользователь открывает WebApp в Telegram
2. Telegram отправляет данные авторизации на `/api/telegram/auth/`
3. Сервер проверяет подлинность данных
4. Создается/находится TelegramUser
5. Если пользователь зарегистрирован — авторизуется
6. Если нет — редирект на регистрацию

## 7. БЕЗОПАСНОСТЬ

### 7.1 Проблемы безопасности

#### ⚠️ КРИТИЧЕСКИЕ:

1. **SECRET_KEY в открытом виде**
   ```python
   SECRET_KEY = 'django-insecure-rx4raz1u)-1$v=)b66qio%2hn_pxu(!*g38dfkgfr0kg#+p*br'
   ```
   **Риск:** Компрометация сессий, CSRF токенов
   **Решение:** Использовать переменные окружения:
   ```python
   SECRET_KEY = os.environ.get('SECRET_KEY')
   ```

2. **TELEGRAM_BOT_TOKEN в открытом виде**
   ```python
   TELEGRAM_BOT_TOKEN = '6599919259:AAGqnQXsjUaXoaSjFqisVy-QmRaDdlgVxdI'
   ```
   **Риск:** Несанкционированный доступ к боту
   **Решение:** Переместить в переменные окружения

3. **DEBUG = True в продакшене**
   ```python
   DEBUG = True
   ```
   **Риск:** Отображение отладочной информации
   **Решение:** 
   ```python
   DEBUG = os.environ.get('DEBUG', 'False') == 'True'
   ```

#### ⚠️ СРЕДНИЕ:

4. **Отсутствие rate limiting**
   - Риск: DDoS атаки, спам регистраций
   - Решение: Django-ratelimit или CloudFlare

5. **Минимальная валидация паролей**
   ```python
   AUTH_PASSWORD_VALIDATORS = [
       {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator',
        'OPTIONS': {'min_length': 4}}
   ]
   ```
   **Риск:** Слабые пароли
   **Решение:** Увеличить минимальную длину до 8+ символов

6. **SQLite в продакшене**
   - Риск: Проблемы с конкурентным доступом
   - Решение: PostgreSQL для продакшена

### 7.2 Хорошие практики безопасности

✅ **CSRF защита:** Включена
✅ **X-Frame-Options:** Настроено
✅ **CSRF_TRUSTED_ORIGINS:** Настроено для домена
